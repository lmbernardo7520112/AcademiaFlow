## 1. Authorization Model

### 1.1 Role Hierarchy
| Role | Scope | Capabilities |
|------|-------|-------------|
| `admin` / `administrador` | Full tenant | CRUD all resources, register users, manage settings |
| `secretaria` | Full tenant | CRUD turmas, alunos, disciplinas, notas; view reports |
| `professor` | Own disciplines only | CRUD notas for own disciplines, view own turmas/alunos |

### 1.2 Ownership Rules
- **Tenant Isolation:** Every query MUST be scoped to `request.user.tenantId`
- **Professor Scope:** A professor can only access resources linked to their `disciplinaIds`
  - `GET /turmas/:id` â†’ permit only if professor teaches a discipline in that turma
  - `POST/PUT/DELETE /notas/:id` â†’ permit only if the nota's discipline belongs to this professor
  - `GET /alunos` â†’ permit only alunos enrolled in turmas where professor teaches
